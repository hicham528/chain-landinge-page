=>===============what is API
واجهة برمجة التطبيقات (API) هي مجموعة من القواعد والبروتوكولات التي تسمح لتطبيقات البرمجيات بالتواصل مع بعضها البعض وتبادل البيانات والوظائف. يمكن اعتبار API كجسر يربط بين التطبيقات المختلفة، مما يسهل عملية التكامل بينها.

كيف تعمل API؟
تعمل API من خلال إرسال طلبات واستقبال استجابات بين 
العميل (التطبيق الذي يرسل الطلب) والخادم (التطبيق الذي يرسل الاستجابة). على سبيل المثال، 
عند شراء منتج عبر الإنترنت واختيار الدفع باستخدام نظام دفع خارجي مثل PayPal، يتم إرسال طلب
 API لاسترداد معلومات الدفع. يتم معالجة هذا الطلب وإرسال الاستجابة من نظام الدفع إلى موقع الويب الأصلي عبر API12.

أنواع API:
واجهات برمجة التطبيقات العامة (Open APIs): متاحة للجميع وتستخدم بشكل شائع لتكامل التطبيقات العامة.
واجهات برمجة التطبيقات الداخلية (Internal APIs): تستخدم داخل المؤسسات لتسهيل التواصل بين الأنظمة الداخلية.
واجهات برمجة التطبيقات الشريكة (Partner APIs): تستخدم لتسهيل التكامل بين شركاء الأعمال.
واجهات برمجة التطبيقات المركبة (Composite APIs): تجمع بين عدة طلبات API في طلب واحد لتبسيط العمليات المعقدة2.
فوائد استخدام API:
تسريع عملية التطوير: يمكن للمطورين استخدام وظائف وخدمات موجودة بدلاً من تطويرها من الصفر.
تحسين الأمان: يمكن مشاركة البيانات الضرورية فقط، مما يحافظ على أمان الأنظمة الداخلية.
التكامل السهل: تسهل API التكامل بين التطبيقات المختلفة، مما يعزز من كفاءة العمل
